About EEG Analysis Dashboard

Research prototype for AI-assisted ADHD detection from EEG signals.

⚠ Important Limitations
  • This is a research prototype, not a diagnostic tool.
  • Trained on 121 children aged 7–12 — performance outside this range is unverified.
  • Requires 19-channel 10-20 montage at 128 Hz or higher; other configurations are not supported.
  • Band power thresholds are indicative, not clinically validated.
  • Always consult a qualified clinician for ADHD diagnosis.

Project Overview

EEG-FM-Bench is a benchmarking framework that compares modern foundation models for EEG against classical approaches in the task of ADHD classification. The system trains and evaluates three architectures on the same dataset under identical conditions, enabling fair comparison.

The dashboard runs all three models in parallel on every uploaded recording and combines their decisions via majority voting, increasing reliability compared to any single model.

Models

  • NeuroGPT — Transformer foundation model with pretrained weights (best performer)
  • EEGPT — Transformer foundation model trained from scratch
  • EEGNet — Compact CNN baseline trained on raw EEG signals

Ensemble Strategy

The dashboard uses a simple majority vote across the three models. Final confidence is the average of the agreeing models' confidences. Reliability is High only when all three models agree with high individual confidence.

Preprocessing Pipeline

  • Notch filter at 50 Hz to remove power-line interference
  • Bandpass filter 0.1–60 Hz
  • Resampling to 256 Hz
  • Non-overlapping windows of 4 seconds (1024 samples)
  • Per-window inference followed by majority vote within each model

Dataset

The benchmark dataset contains EEG recordings from 121 children (61 with ADHD, 60 controls) aged 7–12. Recordings were made using the 10-20 system with 19 channels at 128 Hz, during a visual attention task. ADHD diagnosis was confirmed by an experienced psychiatrist using DSM-IV criteria.

Ethical Considerations

AI-based medical decision support tools must be used responsibly. This system is intended to support clinical research, not to replace clinical judgment.

  • Predictions are based on patterns in training data and may reflect biases
  • The model has not been validated across diverse populations or EEG devices
  • False positives may cause unnecessary anxiety; false negatives may delay needed treatment
  • Patient data should be handled according to applicable privacy regulations (GDPR, HIPAA)
Supported EEG Configurations
Channels (19):
FP1, FP2, F3, F4, C3, C4, P3, P4, O1, O2, F7, F8, T7, T8, P7, P8, FZ, CZ, PZ

Sampling rate: Any (auto-resampled to 256 Hz)
File format: CSV with one column per channel
Reference: Linked earlobes (A1, A2) recommended
Tech Stack
Backend: FastAPI + PyTorch
Models: EEGPT, NeuroGPT, EEGNet (braindecode)
Preprocessing: MNE-Python
Training: Azure GPU cluster
Frontend: Vanilla HTML/JS + Chart.js
